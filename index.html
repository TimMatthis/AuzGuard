Getting Started

AuzGuard provides a fully OpenAI-compatible API that automatically handles compliance, cost optimization, and security for Australian enterprises.

Base URL

https://api.auzguard.com.au/v1

Quick Start

curl -X POST "https://api.auzguard.com.au/v1/chat/completions" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4-turbo",
    "messages": [
      {"role": "user", "content": "Hello, world!"}
    ]
  }'

Authentication

All API requests require authentication using your API key.

API Key

Include your API key in the Authorization header:

Authorization: Bearer YOUR_API_KEY

Getting Your API Key

  1. Sign up for an AuzGuard account
  2. Configure your compliance settings
  3. Navigate to API Keys in your dashboard
  4. Generate a new API key

Chat Completions

Create a completion for the provided chat messages.

Endpoint

POST /chat/completions

Request Body

{
  "model": "gpt-4-turbo",
  "messages": [
    {"role": "system", "content": "You are a helpful assistant."},
    {"role": "user", "content": "Hello!"}
  ],
  "max_tokens": 100,
  "temperature": 0.7,
  "stream": false
}

Error Handling

AuzGuard uses standard HTTP status codes and returns detailed error information.

Error Response Format

{
  "error": {
    "message": "Request blocked due to compliance violation",
    "type": "compliance_error",
    "code": "data_residency_violation",
    "details": {
      "framework": "APRA CPS 234",
      "rule": "RULE-001",
      "audit_hash": "b8c9d0e1f2a3..."
    }
  }
}

Common Error Codes

  • compliance_error - Request violates compliance rules
  • budget_exceeded - Department budget limit reached
  • rate_limit_exceeded - Too many requests
  • invalid_api_key - API key is invalid or expired
